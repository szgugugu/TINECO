# 网络测试后台运行说明

## 修改内容

系统监控应用程序已更新，现在网络测试功能中的ping命令将在后台静默运行，不再显示控制台窗口。

## 技术实现

通过在执行ping命令时添加`creationflags=subprocess.CREATE_NO_WINDOW`参数，使ping.exe进程在后台运行，不显示控制台窗口。

```python
output = subprocess.check_output(
    ["ping", "-n", "4", target],
    stderr=subprocess.STDOUT,
    text=True,
    timeout=10,
    creationflags=subprocess.CREATE_NO_WINDOW  # 隐藏控制台窗口
)
```

## 使用方法

1. 运行系统监控应用程序：
   - 双击`dist/TINECO_System_Monitor.exe`
   - 或双击`启动系统监控.bat`

2. 在网络测试部分，点击"开始测试"按钮
3. 测试过程中不会弹出控制台窗口，测试结果将直接显示在界面中

## 验证方法

运行提供的测试脚本`test_ping_background.py`可以验证ping命令是否在后台运行：

```
C:\Windows\py.exe test_ping_background.py
```

如果测试成功执行且没有显示控制台窗口，说明修改已生效。
