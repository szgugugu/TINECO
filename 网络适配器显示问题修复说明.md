# 网络适配器显示问题修复说明

## 问题描述

用户报告系统监控应用程序中网络适配器部分不显示任何内容。

## 问题原因

原代码中的`update_adapter_info()`函数存在以下问题：

1. 只显示有IP地址的网络适配器，忽略了未连接的适配器
2. 没有处理没有适配器信息的情况

## 解决方案

修改了`update_adapter_info()`函数，主要更改如下：

1. **显示所有网络适配器**：修改了判断条件，只要适配器有MAC地址就显示
2. **处理未连接适配器**：对于没有IP地址的适配器，显示为"未连接"
3. **添加空数据提示**：如果没有找到任何适配器，显示"未找到网络适配器"提示

## 代码变更

```python
# 原代码
# 只添加有IP地址的网络适配器
if ipv4:
    self.adapter_tree.insert("", "end", values=(interface_name, ipv4, mac))

# 修改后代码
# 显示所有网络适配器，即使没有IP地址
if mac:  # 只要MAC地址存在就显示
    if not ipv4:  # 如果没有IP地址，显示为"未连接"
        ipv4 = "未连接"
    self.adapter_tree.insert("", "end", values=(interface_name, ipv4, mac))

# 如果没有适配器信息，添加一条提示信息
if not self.adapter_tree.get_children():
    self.adapter_tree.insert("", "end", values=("未找到网络适配器", "", ""))
```

## 验证方法

1. 运行修改后的系统监控应用程序
2. 检查网络适配器部分是否显示所有适配器
3. 确认未连接的适配器是否显示为"未连接"
4. 确认没有任何适配器时是否显示提示信息

## 技术细节

- 使用`psutil.net_if_addrs()`获取网络接口信息
- 通过检查`socket.AF_INET`和`psutil.AF_LINK`来识别IPv4地址和MAC地址
- 使用`ttk.Treeview`组件显示适配器信息
